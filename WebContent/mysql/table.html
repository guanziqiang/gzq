创建表的标准语句：

create table 表名称(

  列声明1,

  ...

  列声明n,

  primary key\unique\index (列名1，列名2 .....)

)engine = InnoDB, character set uf8 collate utf8_general_ci;

 

查看表的属性：

desc tabl_name;

show table status from lmysql like '%person%';

show full columns from countries;

truncate table_name;属于DDL

修改表结构：

alter table table_name add 列声明

新增加的列在表结构的最后，如果需要制定在某列的后面使用after 列名。

如果需要放到第一位，使用first。

alter table table_name change 需要修改的列名 新的完整列声明(包含列名称)

alter table table_name drop 列名 

把表默认的字符集和所有字符列（CHAR,VARCHAR,TEXT）改为新的字符集：

ALTER TABLE tbl_name CONVERT TO CHARACTER SET character_name [COLLATE ...] 

 修改表属性：

ALTER TABLE tbl_name DEFAULT CHARACTER SET character_name [COLLATE...];

 浮点数

float double decimal

float(m,d)

float和double都会四舍五入（不是我们常用的那种四舍五入）比较混乱不建议使用。

fload（7个有效位） double（16个有效位） 也就是说超出有效位的数值是没有意义的，例如float(11,2) 存一个123456712.98 实际上等同于123456700.00后面的数值已经没有意义了，而且有效位是以最高位优先。

decimal 会保留原来数据，以精度的最后一位来四舍五入。


char(size) size指字符串的最大长度，最多 255 个字符。 

char以固定的宽度填充size，如果填充的字符数小于size末尾一空格符补充。所以如果实际中在末尾存了空格符，取出来的时候Mysql会自动把末尾的空格符去掉。

varchar(size) size指字符串的最大长度。老版本最多 255 个字符。 新版本支持到65535个字节（所以变量的长度=有65534/编码单位）。

变长存储方式，拿出两字节来保存当前存放的字符的长度，所以也解释了为什么存放的长度是65535个字节。 

TINYTEXT 存放最大长度为 255 个字符的字符串。 

TEXT 存放最大长度为 65,535 个字符的字符串。 

BLOB 用于 BLOBs (Binary Large OBjects)。存放最多 65,535 字节的数据。

MEDIUMTEXT 存放最大长度为 16,777,215 个字符的字符串。 

MEDIUMBLOB 用于 BLOBs (Binary Large OBjects)。存放最多 16,777,215 字节的数据。 

LONGTEXT 存放最大长度为 4,294,967,295 个字符的字符串。 

LONGBLOB 用于 BLOBs (Binary Large OBjects)。存放最多 4,294,967,295 字节的数据。 

ENUM(x,y,z,etc.) 允许你输入可能值的列表。可以在 ENUM 列表中列出最大 65535 个值。如果列表中不存在插入的值，则插入空值。 

注释：这些值是按照你输入的顺序存储的。 

可以按照此格式输入可能的值：ENUM('X','Y','Z') 

SET 与 ENUM 类似，SET 最多只能包含 64 个列表项，不过 SET 可存储一个以上的值。
